# OctoAcme Project Management Docs

This repository contains OctoAcme's program and project management process documents.

## Project Management Overview

OctoAcme uses an iterative, customer-focused project management approach with the following key characteristics:

- **Clear Roles**: Program Manager (PM), Product Manager (PdM), Developers, and QA team members each have defined responsibilities
- **Lifecycle Phases**: Initiation → Planning → Execution → Release → Retrospective
- **Lightweight Artifacts**: One-pager, backlog, and risk register guide each project
- **Focus Areas**: Small, testable increments, clear ownership, and continuous improvement through retrospectives

This approach ensures alignment with customer needs, reduces risk through early feedback, and fosters a culture of continuous learning and optimization.

## How to Contribute

To add or update a process document:

1. Submit a pull request updating the corresponding file in `docs/`
2. If adding a new process document, create it in the `docs/` folder and update this README to include a link
3. Update issue templates as needed to reflect new or changed processes
4. Ensure alignment with existing process documentation

For questions about processes or to suggest improvements, please open an issue or contact the PM.
